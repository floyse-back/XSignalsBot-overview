# XSignalsBot

**Інтелектуальна, повністю автоматизована платформа для аналізу криптовалютних сигналів та виконання торгових операцій.**

XSignalsBot — це мікросервісна система, яка приймає ринкові дані в режимі реального часу та сигнали соціальної активності, оцінює їх за допомогою набору кількісних торгових стратегій і автономно виконує угоди на криптовалютній біржі. Платформа також надсилає користувачам сповіщення з рекомендаціями через Telegram-бот, включаючи управління позиціями, контроль ризиків та адміністративну панель.

> 🚀 **Спробувати в дії (Live Demo):** Робочий Telegram-бот доступний за посиланням — **[@XSignalsBot](https://t.me/XSignalsBot)**

## Основні можливості

- **Сканування ринку в реальному часі** — безперервний моніторинг угод з ордер-буку, свічкових даних, подій ліквідації та аномалій тікерів за багатьма криптовалютними парами через WebSocket-потоки біржі.
- **Мультаймфреймовий рушій стратегій** — запуск 12+ незалежних стратегій технічного аналізу (Supertrend, Ichimoku, Bollinger Squeeze, VWAP, Donchian Breakout тощо) на кількох таймфреймах для формування сигналів високої достовірності.
- **AI-аналіз сигналів** — інтегрований шар AI-маршрутизації для збагачення та контекстуалізації згенерованих сигналів перед прийняттям рішень.
- **Автоматичне виконання угод** — відкриття, модифікація та закриття ордерів на Bybit (V5 API) з налаштовуваними take-profit, stop-loss, трейлінг-стопами та динамічним розрахунком розміру позиції.
- **Модуль ризик-менеджменту** — застосування лімітів ризику на угоду та на портфель, розрахунок оптимального розміру позиції, управління паралельними відкритими угодами.
- **Telegram-бот інтерфейс** — надання підписникам сповіщень про сигнали в реальному часі, огляду портфеля, ручного керування угодами та інтегрованого платіжного функціоналу для управління підпискою.
- **Збір соціальних сигналів** — агент-сервіс моніторить кураторські Telegram-канали на предмет аналітичних прогнозів та настроїв спільноти, передаючи ці дані в конвеєр обробки сигналів.
- **Адмін-панель** — захищений REST API з JWT-автентифікацією, рольовим доступом, управлінням користувачами, історією сигналів та відстеженням угод.
- **Фонова обробка завдань** — заплановані та подієво-орієнтовані задачі (агрегування історичних даних, періодичне узгодження угод, переоцінка сигналів) виконуються на розподіленому пулі асинхронних воркерів.
- **CI/CD конвеєр** — автоматична збірка та деплой через GitHub Actions із self-hosted ранером та Docker Compose.

## Архітектура системи

XSignalsBot побудований на **подієво-орієнтованій мікросервісній архітектурі**, де кожен сервіс комунікує асинхронно через центральний брокер повідомлень. Дані проходять через багатоступеневий конвеєр: сирі ринкові дані спочатку скануються, потім аналізуються стратегіями, далі оркеструються бекендом і, нарешті, виконуються торговим ботом.

```

│                      ЗОВНІШНІ ДЖЕРЕЛА                            │

             │                                  │
    ┌────────▼────────┐                ┌────────▼────────┐
    │   XScreeners    │                │   Agent Bot     │
    │ (Сканер ринку)  │                │ (Збирач соц.    │
    │                 │                │   сигналів)     │
    └────────┬────────┘                └────────┬────────┘
             │ data_stream                      │ signals /
             │                                  │ channel_status
    ┌────────▼────────┐                         │
    │  XStrategies    │                         │
    │ (Рушій стратегій │──────────┐             │
    │  та генерація   │ signals / │             │
    │  сигналів)      │ upsert_   │             │
    └─────────────────┘ sources   │             │
             ┌────────────▼───────▼─────────────▼──┐
             │          Redis (Кеш-шар)             │
             │       RabbitMQ (Брокер повідомлень)  │
             └────────────────┬────────────────────┘
                              │
              ┌───────────────▼────────────────┐
              │         Backend API            │
              │  (Ядро оркестрації, Auth,      │
              │   Обробка сигналів, AI)        │
              └──┬──────────┬──────────┬───────┘
                 │          │          │
      get_signals│  crypto_ │  update_ │
        _{group} │  signal  │  chats   │
                 │          │          │
     ┌───────────▼───┐  ┌──▼──────────▼──┐
     │   Trade Bot   │  │  Telegram-бот  │
     │ (Виконання    │  │ (Сповіщення    │
```

### Огляд сервісів

| Сервіс | Відповідальність | Комунікація |
|---|---|---|
| **XScreeners** | Підключення до WebSocket та REST-потоків біржі, виявлення ринкових аномалій та сплесків обсягів, кешування знімків у Redis | Публікує сирі події → **XStrategies** |
| **XStrategies** | Споживає дані сканера, виконує мультаймфреймовий технічний аналіз (12+ стратегій), продукує рейтинговані торгові сигнали | Підписується на **XScreeners**; публікує сигнали → **Backend** |
| **Agent Bot** | Сервіс на базі Telethon, що моніторить Telegram-канали для збору соціальних та аналітичних криптосигналів і статусу каналів | Публікує сигнали та дані каналів → **Backend** напряму |
| **Backend API** | Центральний оркестратор — обробка сигналів, AI-збагачення, JWT-автентифікація, управління користувачами, координація життєвого циклу ордерів | Приймає від **XStrategies** та **Agent Bot**; маршрутизує → **Trade Bot** та **Telegram** |
| **Trade Bot** | Отримує оброблені ордери, розраховує розмір позиції, розміщує та керує ордерами на Bybit (демо- та реальний режим) | Приймає від **Backend**; публікує результати виконання → **Backend** |
| **Telegram-бот** | Користувацький інтерфейс на aiogram — сповіщення про сигнали, огляд портфеля, підписні платежі | Отримує сповіщення від **Backend** |
| **TaskIQ Воркери** | Розподілені асинхронні воркери для запланованих задач (синхронізація даних, оновлення угод, періодичні перерахунки) | Працюють у контексті **Backend** |
| **Nginx** | Зворотний проксі-сервер, що маршрутизує зовнішній трафік до Backend API | Фронтить **Backend API** |

## Технологічний стек

| Категорія | Технологія |
|---|---|
| **Мова програмування** | Python 3.10+ |
| **Веб-фреймворк** | FastAPI, Uvicorn |
| **ORM / База даних** | SQLAlchemy 2.0 (асинхронна), PostgreSQL 17, Alembic (міграції) |
| **Брокер повідомлень** | RabbitMQ 4.x (через FastStream, aio-pika) |
| **Кеш** | Redis 7.2 |
| **Черга завдань** | Taskiq (з планувальником та воркер-процесами) |
| **Telegram-бот** | aiogram 3.x |
| **Монітор каналів** | Telethon (Telegram MTProto-клієнт) |
| **Біржовий API** | pybit (Bybit V5 REST та WebSocket) |
| **Технічний аналіз** | pandas, pandas-ta, numpy |
| **AI-інтеграція** | Google Generative AI, Groq |
| **Зворотний проксі** | Nginx (Alpine) |
| **Контейнеризація** | Docker, Docker Compose |
| **CI/CD** | GitHub Actions (self-hosted ранер) |
| **Моніторинг** | Sentry SDK, RedisInsight, pgAdmin |

## Структура проєкту

```
XSignalsBot/
├── backend/          # Ядро FastAPI-сервісу (REST API, автентифікація, оркестратори)
├── telegram/         # Telegram-бот (користувацький інтерфейс на aiogram)
├── agent_bot/        # Збирач соціальних сигналів (Telethon-монітор каналів)
├── xscreaners/       # Сканер ринкових даних у реальному часі (Bybit WebSocket)
├── xstrategies/      # Мультаймфреймовий рушій стратегій та сигналів
├── trade_bot/        # Виконання угод та ризик-менеджмент (Bybit V5 API)
├── nginx/            # Конфігурація зворотного проксі
├── alembic/          # Скрипти міграцій бази даних
├── .github/          # CI/CD конвеєр (GitHub Actions)
└── docker-compose.yml
```
